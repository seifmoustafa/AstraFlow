# Code of Conduct

AstraFlow is an open source project for .NET developers. Participation in the project should stay technical, respectful, and focused on making the package family useful and maintainable.

## Expected Behavior

- Be direct and constructive.
- Assume public comments, issues, and pull requests are read by people with different experience levels.
- Keep disagreements focused on code, design, documentation, packaging, and user impact.
- Respect maintainers' time by using clear reproduction steps and scoped pull requests.
- Avoid harassment, personal attacks, discriminatory language, or repeated off-topic comments.

## Unacceptable Behavior

The following is not acceptable in project spaces:

- Threats, harassment, stalking, or intimidation.
- Insults aimed at a person or group rather than a technical decision.
- Sexualized language or imagery.
- Discriminatory comments related to identity, background, or ability.
- Publishing private information without explicit permission.
- Repeated disruption after a maintainer asks for a thread to return to topic.

## Reporting

Report conduct concerns privately using the contact path in [SECURITY.md](https://github.com/seifmoustafa/AstraFlow/blob/main/SECURITY.md) when the concern is sensitive, or open a GitHub issue when the concern can be handled publicly.

Maintainers may remove comments, close issues or pull requests, block accounts, or take other moderation action when needed to keep the project usable.

## Scope

This code applies to AstraFlow GitHub issues, pull requests, discussions, release comments, and other project-maintained spaces.
