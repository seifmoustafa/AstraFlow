# Support

Use the smallest public channel that fits the problem.

## Maintenance Support

AstraFlow is MIT-licensed and free to use. If the package saves you time and you want to support ongoing maintenance, you can use:

- Buy Me a Coffee: https://buymeacoffee.com/seifmoustafa

## Usage Questions

Open a GitHub issue for questions about:

- choosing between `AstraFlow`, `AstraFlow.Mediator`, `AstraFlow.Mapper`, `AstraFlow.Diagnostics`, `AstraFlow.Testing`, and `AstraFlow.Contracts`,
- registration behavior,
- mapping or projection validation,
- diagnostics output,
- package installation and target framework support.

Include the package ID, package version, target framework, and a small code sample when possible.

## Bugs

Open a GitHub issue with:

- the package ID and version,
- the target framework,
- the expected behavior,
- the actual behavior or exception message,
- a minimal reproduction or failing test.

## Security

Do not open a public issue for a suspected vulnerability. Follow [SECURITY.md](https://github.com/seifmoustafa/AstraFlow/blob/main/SECURITY.md).

## Releases

Release history is tracked in [CHANGELOG.md](https://github.com/seifmoustafa/AstraFlow/blob/main/CHANGELOG.md). Publishing steps are documented in [docs/publishing.md](https://github.com/seifmoustafa/AstraFlow/blob/main/docs/publishing.md).
